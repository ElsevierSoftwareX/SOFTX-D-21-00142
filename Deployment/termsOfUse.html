<h1>Description</h1> MicroVIP is part of European project PROCHIP, you can find more information about it and this project at <a href="https://www.creatis.insa-lyon.fr/site7/en/PROCHIP">https://www.creatis.insa-lyon.fr/site7/en/PROCHIP</a>. It consists in a microscopy image simulation tool for following microscopy techniques: wide-field, confocal, Structured Illumination Microscopy (SIM), Light Sheet Microscopy (LSM), balanced Super-resolution Optical Fluctuation Imaging (bSOFI) and Stochastic Optical Reconstruction Microscopy (STORM). This tool is divided in three modules, respectively modeling of ground truth cells (3D coordinates of biomarkers), simulation of image acquisition process, and features extraction. <h2>Parameters and outputs</h2> <h3>Number of cells</h3> MicroVIP allows generation and imaging of any given number of cells in parallel. They are generated with the same parameters but using random distributions to create variability, and each one is statistically independant from the others. <h3>Pipeline</h3> This parameter allows execution of aforementioned modules in a single pipeline. If all three modules should be executed, set pipeline=2. This corresponds to cells generation, followed by simulation of these cell's microscopy images, followed by features extraction on these images. All intermediate files (output of each module) will be available in the result directory. </br>Setting pipeline=1 stops the pipeline after module 2: micrscopy images are simulated but features extraction is not performed, saving computation time. </br>To perform only cell generation, set pipeline=1. <h3>Configuration file</h3> This should be the path to a .ini file containing all parameters for the simulation. Examples are available in File Transfer under /vip/Prochip (group)/data/. To customize an execution, download one of these examples and modify parameter values as needed. Then, upload the new .ini file to desired location in File Transfer, and enter its path in the interface (you can browse File Transfer to find the file thanks to the magnifying glass button). <h3>Outputs</h3> At the end of the execution, entered results directory will contain an archive microVipOut.tar, organized in sub-directories containing results of each module. 'markers_coordinates' contains three columns .csv files of 3D biomarkers coordinates, output of module 1. 'ground_truth_images' and 'final_images' respectively contain binarized ground truth 3D images of biomarkers, and simulated microscopy images, all in .tif format. These are outputs of module 2, available only if pipeline>0. 'extracted_features' contains .json files of extracted features for each image, output of module 3 and available only if pipeline=2.</br> In addition, results directory contains one .tar archive for each cell, which name starts with 'XofY_' where X is concerned cell and Y is the total number of cells generated. This archive contain the same aforementionned files, but only for one generated cell. <h1>Terms of use</h1> MicroVIP is an open-source application, you can find license information and source code at: <i>Link coming soon</i>.</br> It is partially based on following works, used, modified and/or deployed by Guillaume Vanel as permitted by the terms of hereafter license notices.
